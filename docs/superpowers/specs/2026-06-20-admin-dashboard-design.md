# 智慧通勤后台管理界面设计稿

## 1. 目标

本次设计面向 `fronted/` 管理后台 Web，目标是将当前偏“文字堆砌”的演示页重构为一套更接近真实运营后台的管理界面。

核心目标如下：

- 首页具备后台驾驶舱感，能够一眼看到运营状态
- 候车热力图使用 ECharts 实现，不再手写热区块
- 地图调度看板作为核心工作区之一，支持线路和站点联动
- 常用管理模块具备表格、筛选、状态、详情等真实后台交互
- 全部数据采用固定演示数据，但界面必须像真实后台

## 2. 设计范围

本次仅覆盖后台 Web 界面设计与前端演示交互，不涉及真实接口、权限系统或数据库。

范围内页面：

- 首页看板
- 地图调度看板
- 候车热力图
- 用户管理
- 司机管理
- 车辆管理
- 线路管理
- 站点管理
- 班次 / 任务管理
- 一键呼叫管理
- 订单管理
- 退款管理
- 优惠券管理
- 乘车次数权益管理
- 共享电动车相关管理模块

范围外内容：

- 后端接口实现
- 数据持久化
- 登录认证流程
- 真实地图服务接入
- 生产级权限控制

## 3. 页面结构

### 3.1 总体布局

后台采用标准管理后台布局：

- 左侧固定菜单栏
- 顶部工具条
- 中间主内容区
- 右侧辅助分析区或快捷操作区

这样可以让页面呈现出真实管理后台的工作流，而不是单纯数据展示页。

### 3.2 首页看板

首页作为运营驾驶舱，主要展示四类内容：

1. 核心 KPI
2. 趋势图表
3. ECharts 候车热力图
4. 地图调度看板

首页需要避免“只有描述，没有数据”的问题，因此每个模块都必须带有图形化信息，例如柱状图、折线图、热力图、地图叠层和状态卡片。

## 4. 核心模块

### 4.1 首页驾驶舱

首页顶部展示四个核心指标：

- 在线车辆
- 今日呼叫
- 退款待审
- 资格待审

中部展示趋势图，建议包含：

- 近 7 天候车呼叫趋势
- 订单变化趋势
- 退款审核趋势

下方展示两个核心工作区：

- 左侧：地图调度看板
- 右侧：ECharts 候车热力图

首页右侧保留实时摘要、待办列表和关键告警，让页面看起来像运营中心而不是普通列表页。

### 4.2 候车热力图

热力图必须使用 ECharts 实现，采用热力图或散点热区叠层，不再用手写颜色块。

热力图支持：

- 按时段切换
- 按线路切换
- 按站点筛选
- 点击高热点联动地图调度和右侧详情

展示内容建议包括：

- 站点名称
- 热度值
- 时间段
- 热度等级
- 调度建议

### 4.3 地图调度看板

地图调度看板用于展示：

- 车辆位置
- 线路轨迹
- 站点标记
- 当前线路状态
- 车辆空座/满载情况

地图调度区应提供真实后台常见操作：

- 切换线路
- 聚焦某辆车
- 查看站点详情
- 发送调度提醒
- 标记异常

地图本体可以继续沿用当前项目中的固定地图演示逻辑，但视觉上要更像后台工作台，而不是静态展示。

### 4.4 运营管理模块

常用后台模块不做纯说明页，而是尽量做成“能操作”的管理界面。

每个模块统一包含：

- 页面标题
- 筛选区
- 状态标签
- 表格或卡片列表
- 详情抽屉 / 弹窗
- 基础操作按钮

典型模块示例：

- 用户管理：用户列表、资格状态、乘车记录入口
- 司机管理：司机档案、绑定车辆、评价、考勤
- 车辆管理：车辆状态、座位数、维护状态
- 线路管理：线路颜色、站点顺序、启停状态
- 班次 / 任务管理：今日任务、运行中、待出发、异常
- 退款管理：待审、通过、驳回
- 共享电动车管理：车辆、围栏、资格、实操审核

## 5. 交互设计

### 5.1 菜单交互

左侧菜单点击后切换模块内容，保留当前选中态，支持层级清晰的导航感。

### 5.2 图表交互

图表区域支持：

- 时段切换
- 数据筛选
- 点击图表项查看详细数据
- 导出当前视图的演示按钮

### 5.3 地图联动

热力图和地图调度之间需要联动：

- 点击热力图站点后，地图自动高亮对应站点
- 切换地图线路后，热力图可同步聚焦对应线路
- 右侧摘要区显示当前选中对象的详情

## 6. 视觉风格

后台采用亮主题、企业中台风格：

- 白底或极浅底色
- 蓝色为主强调色
- 绿 / 橙 / 红用于状态区分
- 图表和卡片采用清晰边界
- 避免大段纯文字堆叠
- 避免过重装饰，保持管理后台专业感

## 7. 数据演示原则

虽然本次只做界面演示，但每个核心区域都必须有“像真的”数据：

- KPI 有数值
- 图表有趋势
- 热力图有热区
- 地图有位置
- 表格有状态
- 操作有反馈

禁止继续采用只有标题、没有内容的空壳模块。

## 8. 验证方式

完成后需要至少验证以下内容：

- 页面能正常打开
- 图表区域能显示 ECharts
- 热力图能按时段切换
- 地图调度和热力图能联动
- 左侧菜单切换正常
- 页面不再出现大面积纯文字堆砌

## 9. 非目标

本次不做：

- 真实后端接口对接
- 真实登录权限
- 真实地图 SDK 深度接入
- 完整报表导出
- 复杂动画或炫技式大屏

## 10. 结论

本次后台重构应以“企业中台 + 运营驾驶舱”为主线，优先把首页看板、ECharts 热力图和地图调度做实，再补齐常用管理模块的真实后台交互。
